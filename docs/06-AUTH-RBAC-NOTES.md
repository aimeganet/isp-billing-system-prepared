# ملاحظات Auth و RBAC

## ما الموجود الآن؟
- جلسات محلية عبر جدول `Session`
- مستخدمون عبر جدول `User`
- أدوار عبر جدول `Role`
- صلاحيات عبر جدول `Permission`
- ربط متعدد بين المستخدمين والأدوار عبر `UserRole`
- ربط متعدد بين الأدوار والصلاحيات عبر `RolePermission`
- دور أساسي legacy داخل `User.role` ما زال موجودًا كطبقة توافق ودعم سريع

## كيف تعمل الصلاحيات؟
1. المستخدم يسجل الدخول عبر البريد وكلمة المرور
2. تنشأ Session في قاعدة البيانات ويُحفظ token في Cookie محلية
3. `lib/auth.ts` يقرأ الجلسة والمستخدم الحالي
4. `lib/permissions.ts` يبني permission set من:
   - الدور الأساسي داخل `User.role`
   - أي أدوار إضافية عبر `UserRole`
5. الصفحات والإجراءات تستدعي `requirePermission(...)`

## الأدوار النظامية الافتراضية
- `admin`
- `supervisor`
- `employee`

## ماذا يفعل الـ seed؟
- ينشئ الصلاحيات الأساسية
- ينشئ الأدوار النظامية
- يربط كل دور بصلاحياته الافتراضية
- ينشئ مستخدم المدير الافتراضي
- يربط المدير بدور `admin`

## بيانات الدخول الافتراضية
- البريد: `admin@local.test`
- كلمة المرور: `ChangeMe123!`

## ملاحظات مهمة
- هذه طبقة محلية مناسبة للبداية والبرنامج المكتبي
- يمكن لاحقًا استبدالها أو توسيعها بمصادقة أقوى أو 2FA
- لم تتم إضافة حماية CSRF متقدمة أو rate limiting بعد
- لا يوجد حتى الآن reset password أو invite flow
